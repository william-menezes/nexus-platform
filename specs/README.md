# Specs

Cada feature deve ter uma pasta em `specs/` contendo pelo menos:

- `spec.md` (o que/por quê + critérios de aceite)
- `plan.md` (como vamos fazer)
- `tasks.md` (decomposição em tarefas)
- `checklist.md` (checagens de entrega)

Convenção de nome:

- `specs/YYYY-MM-DD-<slug>/`

Criação rápida:

```bash
npm run spec:new -- retorno-de-vendas --title "Devolução de vendas"
```
